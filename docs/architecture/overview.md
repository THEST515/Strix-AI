# 架构说明

## 目标

围绕 `usestrix/strix` 构建一个适合课程展示的 AI 辅助安全分析平台原型，用于授权训练场景。

## 核心原则

不要把 Strix 内部逻辑直接嵌入平台，而是把它当成独立扫描引擎，由后端调度调用。

## 主流程

1. 用户在前端创建扫描任务
2. 后端保存任务并校验边界说明
3. 后端构造 Strix 命令
4. 后端以非交互模式调用 Strix
5. 后端读取 Strix 产物目录
6. 解析器把结果统一成平台内部结构
7. AI 摘要服务生成概览、重点风险和修复建议
8. 前端渲染报告并支持导出

## 模块边界

### 前端

- 任务创建表单
- 任务列表
- 报告详情页
- 导出按钮

### 后端 API

- 创建任务
- 查询任务
- 运行任务
- 读取结果
- 读取摘要
- 导出报告

### Strix 运行层

- 命令构造
- 子进程调用
- 状态跟踪
- 运行目录管理

### 结果解析层

- 读取原始结果
- 映射为统一风险结构
- 统一标题、严重级别、证据与修复建议

### AI 摘要层

- 汇总扫描结果
- 识别重点风险
- 输出修复建议摘要

## 为什么不直接嵌入 Strix 内部实现

1. 耦合度太高
2. 后续升级困难
3. 不利于课程项目解释
4. 4 人协作时职责边界不清晰

## MVP 稳定性策略

1. 优先准备一个稳定的演示目标
2. 预留夹具回放模式作为保底
3. 报告导出先做简单版本
4. 持久化先使用本地最小方案
