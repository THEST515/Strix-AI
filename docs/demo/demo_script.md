# 演示脚本

## 1. 开场口径

一句话介绍：

> 这是一个基于 Strix 的 AI 辅助安全分析平台课程演示型 MVP，重点不是堆架构，而是把真实扫描、运行态解释、中文风险结果和报告导出打通。

## 2. 演示顺序

### 第一段：稳定演示

1. 新建 `fixture` 任务
2. 展示任务创建、任务切换、摘要区、风险详情区
3. 导出 Markdown
4. 导出 DOCX

讲解口径：

- 这段用于稳定展示产品流程
- 不依赖 Docker 和 Strix 现场环境

### 第二段：真实扫描演示

1. 新建 `latest_real_run` 任务
2. 选择 `3 / 5 / 10 分钟` 时长
3. 点击 `启动真实 Strix 扫描`
4. 讲解执行轨迹区

优先讲 6 个字段：

- 当前阶段
- 攻击面概览
- LLM 请求量
- 收敛状态
- 失败分类
- 下一步建议

建议口径：

- 现在关注点不再是“Strix 能不能被调用”
- 而是“扫描走到了哪一阶段、是否收敛到漏洞证据、下一步该继续还是收缩”

### 第三段：结果与导出

如有 findings：

1. 打开风险详情
2. 强调结果已中文化
3. 导出 Markdown / DOCX

如没有 findings：

1. 解释这不等于“系统没工作”
2. 结合运行态解释：
   - 是否仍在侦察
   - 是否攻击面不足
   - 是否超时未收敛
   - 是否运行环境异常

## 3. 关键卖点

- 真实 Strix 扫描链路没有回退
- 扫描运行态可解释
- findings 支持中文化与缓存，避免结果反复跳版本
- 失败或超时后，已发现漏洞尽量保留
- 可直接导出固定模板 DOCX

## 4. 收尾口径

推荐收尾：

> 这个 MVP 已经证明了三件事：第一，Strix 真实被调用；第二，平台能解释扫描运行态；第三，已有结果可以沉淀为中文报告并导出。下一步重点不是大重构，而是继续提升真实扫描收敛质量和演示稳定性。
